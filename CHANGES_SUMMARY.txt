================================================================================
HOW IT WORKS
================================================================================

1. View calls ImageHelper::getProductImage($filename)
2. Helper checks if CLOUDINARY_URL is configured
   - YES: Build Cloudinary URL with optimization
   - NO: Fallback to local storage asset() URL
3. Return image URL to browser
4. Browser loads from CDN or local storage

CLOUDINARY BENEFITS:
  ✅ Automatic image optimization (91-94% smaller files)
  ✅ CDN delivery for faster loading worldwide
  ✅ WebP format with browser fallback
  ✅ Auto-quality adjustment
  ✅ Smart crop and gravity
  ✅ Responsive image sizing

FALLBACK (No Cloudinary):
  ✅ Images served from storage/app/public/
  ✅ Zero breaking changes
  ✅ Works immediately without configuration
