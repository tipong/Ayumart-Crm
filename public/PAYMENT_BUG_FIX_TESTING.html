🧪 Testing Guide: Payment Bug Fix & Auto-Cancel

Comprehensive testing procedures for payment system improvements

✅ Test 1: Normal Payment Flow

Objective

Verify that standard payment flow works correctly with proper status updates and stock reduction

Steps

  1. Login as customer and add items to cart
  2. Go to checkout page
    • Select "Dikirim Kurir" as shipping method
    • Select a delivery address
    • Note the shipping cost (Ongkir) displayed
  3. Complete checkout and proceed to payment
    • Verify payment page shows same Ongkir amount
    • Verify total = subtotal - diskon + ongkir + membership fee
  4. Complete payment in Midtrans test mode
    • Use test card: 4811 1111 1111 1114
    • Any future date, any CVV
  5. Wait for page to redirect or check /orders page
    • Order status should be "Sudah Bayar" ✓

Expected Results

✓ Order status updates to "Sudah Bayar"

✓ Shipping cost consistent on all pages

✓ Membership points awarded

✓ Stock reduced in database integrasi

✓ Shipment record created

Logs to Check

tail -f storage/logs/laravel.log | grep -i "payment\|callback"

Look for: "✅ Midtrans Callback Received", "✅ Payment successful"

⏳ Test 2: Payment Callback Delayed

Objective

Verify that payment status updates correctly even if Midtrans callback is delayed, via polling mechanism

Steps

  1. Complete payment in Midtrans (as in Test 1)
  2. Don't close browser, leave payment page open
  3. Frontend will poll /payment-status/{order_id} every 3 seconds
  4. Observe browser console logs:
    • Should see "🔄 Checking payment status..."
    • After callback arrives: "✅ Payment confirmed on server!"
  5. Page should redirect to /orders within 2 seconds

Expected Results

✓ Status updates via polling (no manual page refresh needed)

✓ Page auto-redirects to orders

✓ Order shows as "Sudah Bayar"

Browser Console Check

Open F12 → Console, look for:

🔄 Checking payment status... ✅ Payment confirmed on server!

🚫 Test 3: Auto-Cancel Expired Orders

Objective

Verify that unpaid orders are automatically cancelled after 15 minutes

Setup

Create an order without paying:

  1. Complete checkout but close Midtrans popup (don't pay)
  2. Get the order code from /orders page, e.g., "TRX-20260321-abc1"

Steps

  1. In database, simulate expired payment:
    UPDATE tb_transaksi SET payment_expired_at = DATE_SUB(NOW(), INTERVAL 1 MINUTE) WHERE kode_transaksi = 'TRX-20260321-abc1';
  2. Run auto-cancel command:
    php artisan orders:cancel-expired --verbose
  3. Check order status in database:
    SELECT kode_transaksi, status_pembayaran, payment_expired_at FROM tb_transaksi WHERE kode_transaksi = 'TRX-20260321-abc1';

Expected Results

✓ status_pembayaran = 'kadaluarsa'

✓ Stock restored to database integrasi

✓ Log entry: "✅ Order automatically cancelled"

Check Logs

tail -f storage/logs/laravel.log | grep "CancelExpiredOrders"

Look for: "Order automatically cancelled due to payment timeout"

🔄 Test 6: Stock Management

Objective

Verify that stock is correctly reduced after payment and restored on cancellation

Steps

  1. Note initial stock in database integrasi:
    SELECT id_produk, id_cabang, total_stok FROM tb_stok_cabang WHERE id_produk = 1 AND id_cabang = 1;
  2. Add product to cart and complete payment
  3. Check stock after payment:
    SELECT total_stok FROM tb_stok_cabang WHERE id_produk = 1 AND id_cabang = 1;

    Should be reduced by qty purchased

  4. Cancel the order and check stock restoration:
    UPDATE tb_transaksi SET status_pembayaran = 'kadaluarsa' WHERE id_transaksi = 1;

    Or run: php artisan orders:cancel-expired

  5. Verify stock restored:
    SELECT total_stok FROM tb_stok_cabang WHERE id_produk = 1 AND id_cabang = 1;

    Should be same as initial

Expected Results

✓ Stock reduced after payment

✓ Stock restored on cancellation

✓ Stock never goes negative

✓ Log entries for all stock changes

Check Logs

tail -f storage/logs/laravel.log | grep -i "stock\|stok"

📋 Test 7: Membership Points

Objective

Verify that membership points are awarded after successful payment

Steps

  1. Create order for customer without membership
  2. Note customer's current points and tier:
    SELECT user_id, points, tier FROM tb_membership WHERE user_id = 1;
  3. Complete payment for order
  4. Check points awarded:
    SELECT points, tier FROM tb_membership WHERE user_id = 1;

    Points = Total / 20000 (rounded)

  5. Verify tier upgraded if applicable

Expected Results

✓ Points awarded: order_total / 20000

✓ Tier upgraded if threshold reached

✓ New membership created if first transaction

✓ Log entries for points calculation

Check Logs

tail -f storage/logs/laravel.log | grep "points\|membership"

Look for: "Membership points awarded", "New membership created"

🔧 Troubleshooting

❌ Order not updating after payment

  • Check if callback URL is configured
  • Verify Midtrans can reach your callback endpoint
  • Check /payment-status API works
  • Verify Midtrans keys in .env

❌ Auto-cancel not working

  • Verify cron is running: check logs
  • Run manually: php artisan orders:cancel-expired
  • Check scheduler: php artisan schedule:list
  • Verify database connection

❌ Ongkir different on pages

  • Check hidden input #shipping_cost set
  • Verify distance calculation correct
  • Check form submitted with value
  • Verify database stores correct ongkir

❌ Stock not restored

  • Verify IntegrasiProdukService available
  • Check db_integrasi connection configured
  • Verify tambahStok() method exists
  • Check logs for errors

❌ Timer not showing

  • Check payment_expired_at is set
  • Verify JavaScript loaded
  • Check element IDs exist
  • Check browser console for errors

❌ Database errors

  • Run migration: php artisan migrate
  • Check columns exist: DESCRIBE tb_transaksi
  • Verify indexes created
  • Check database connection
