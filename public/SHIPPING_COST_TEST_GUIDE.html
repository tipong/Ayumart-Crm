🚀 Shipping Cost Bug Fix

Testing Guide & Verification Checklist

📋 Problem Summary

Bug: Shipping cost (ongkir) was correct on checkout page but reverted to Rp 15,000 on payment page

Root Cause: Payment page displayed total from local controller variable instead of fresh database data

Status: ✅ FIXED

🧪 Test Scenario 1: Kurir Delivery with Distance

1️⃣Navigate to Checkout

Go to cart and proceed to checkout page

2️⃣Select Kurir Delivery

Choose "Dikirim Kurir" option

3️⃣Select Delivery Address

Select an address with valid coordinates (latitude, longitude)

Expected: Shipping cost should be calculated based on distance

Formula: Distance (km) × Rp 10,000

4️⃣Verify Checkout Page Total

Check order summary on the right:

Component Expected Behavior
Subtotal Sum of all products
Diskon Membership Shown if user has active membership
Ongkos Kirim Distance-based cost (not Rp 15,000)
Total All components correctly summed

5️⃣Submit Checkout Form

Click "Lanjut ke Pembayaran" button

Verify: No validation errors

6️⃣🔴 CRITICAL: Check Payment Page Total

This is where the bug was occurring

Expected:

  • Large total at top should match checkout total ✅
  • Should NOT show Rp 15,000 if distance is different ❌
  • "Ongkos Kirim" in breakdown should show actual distance cost

7️⃣Verify Payment Breakdown

Check "Rincian Pembayaran" card:

Subtotal Produk : Rp XX.XXX Diskon Membership : -Rp X.XXX (if applicable) Ongkos Kirim : Rp XX.XXX (distance-based) ✅ ───────────────────────── Total Pembayaran : Rp XX.XXX

⚠️ Verification Points

  • Payment page total = Checkout page total ✅
  • Ongkir value matches distance calculation ✅
  • No "Rp 15,000" fallback shown ✅
  • Midtrans popup shows correct amount ✅
